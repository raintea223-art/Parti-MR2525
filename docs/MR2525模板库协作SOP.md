## 1. 目标与原则

本 SOP 定义团队如何将 SketchUp 方案模板化：对外快速报价，对内指导生产。

| 原则 | 说明 |
|------|------|
| **固定尺寸** | 每个模板对应一套确定的宽 × 深 × 高（mm），不做参数化变体 |
| **纯手工 BOM** | 从 skp 人工录入清单；短期内不依赖 SketchUp 自动导出 |
| **一份清单** | 数量、规格、尺寸只有一份；对内成本与对外报价仅**单价口径**不同 |
| **皮肤 = 板材选配** | 结构由 MR2525 型材体系承担；皮肤在模板库中主要体现为**板材 BOM** |
| **有就上架** | 模板完成一个录入一个，不追求一次性批量冲刺 |
| **齐全才发布** | 发布 checklist 任一项缺失，不得进入「已发布」状态 |
| **skp 先行** | 录入时通常已有成品 skp；**上传 skp 是新建模板后的第一步** |
| **统一资产目录** | 每个模板在 `data/uploads/{模板编号}/` 下集中存放 skp 与全部图片；**不再**向 NAS `Parti template model` 同步 skp |
| **飞书对外** | 客户通过**飞书多维表格 · 画册视图**浏览已发布方案，而非逐条复制文字摘要 |

## 2. 模板是什么

每个「模板」= 一个可对外销售的**固定尺寸单品方案**，包含三层信息：

```
模板（固定尺寸）
├── 结构层：MR2525 型材 / 六通 / 无时序连接件 / 脚件 / 结构五金
├── 板材层：可有多行（不同材质、不同厚度可并存）
│     └── 每行独立：类型 + 厚度 + 标配颜色 + 尺寸 + 数量
└── 对外包装：封面 / 实拍·效果图·渲染图 / 参考价 / 报价口径 / 一句话卖点
```

- **结构 BOM** 与皮肤无关，所有皮肤档位共用同一套结构清单。
- **板材 BOM** 可有多行。例如：顶板用免漆板 12mm，侧板用海洋板 18mm——各占一行。
- **图片**分三类上传（见 §12.3）：实拍照片（可选）、效果图（必填）、渲染图（必填）；**封面**由系统按规则自动选取，不单独作为上传类别。
- 若产品需要完全不同的材质/厚度组合逻辑，应登记为**不同模板**，而不是在同一模板内用「升级」切换材质或厚度。

## 3. 皮肤选配规则

### 3.1 「升级」的含义

对**某一行**板材，「升级」**仅允许在同材质、同厚度内更换颜色**：

| 允许 | 不允许 |
|------|--------|
| 免漆板 12mm · 白色 → 免漆板 12mm · 木色 | 免漆板 → 海洋板（换材质） |
| 海洋板 18mm · 木色 → 海洋板 18mm · 白色 | 12mm → 25mm（换厚度） |

多行板材模板中，**每一行独立遵守上述规则**。升级行 A 的颜色不影响行 B 的材质与厚度。

### 3.2 「可定制」开关（系统字段 `skin_upgrade_enabled`）

| 状态 | 何时使用 | 对外表现 |
|------|----------|----------|
| **关（默认）** | 新模板上传、供应链未就绪时 | 只报**标配皮肤**；参考价为**单一价位**（下限 = 上限） |
| **开** | 单价库已维护同色厚异色项、供应链成熟后 | 可报**皮肤价差区间**；可选色以 **板材 BOM + 单价库** 为准 |

**初期流程**：所有新模板 **可定制 = 关**。待单价库补全、审核员确认后，再在详情基本信息栏开启并更新参考价区间。

详情页以 **chip 复选框**（文字与勾选框一体）呈现，文案为「可定制」。

### 3.3 字段分工

| 字段 | 用途 |
|------|------|
| **板材 BOM 行** | 每行填写实际标配的材质、厚度、颜色、长宽、数量（**新模板皮肤信息以 BOM 为准**） |
| **皮肤选配说明** `panel_note` | **历史模板**可有内容（图册/PDF 仍展示）；**新模板不再在详情维护** |

**替换行原则**：报价与生产以 BOM 中**当前标配行**为准。客户选色 = 将该行颜色替换为允许列表中的某一色，**数量与尺寸不变**。

## 4. 定价口径

### 4.1 对外报价（给客户 / 参考价）

```
型材对外单价 = ROUNDUP(长度(in) × 1.778 + 42, 0)

六通 / 五金 / 板材对外单价 = 单价库「对外价」

物料对外小计 = Σ(各行数量 × 对外单价)

对外加工费   = 物料对外小计 × 10%

对外总报价   = 物料对外小计 × 1.10
```

> 对外加工费统一按**全部物料小计**的 10% 计算，包含型材、六通、五金、板材。

### 4.2 对内成本（生产 / 核算）

```
型材对内单价 = 型材对外单价 × 0.5

六通 / 五金 / 板材对内单价 = 单价库「对内价」

对内成本 = Σ(各行数量 × 对内单价)
```

> **对内不含加工费**（10% 加工费仅体现在对外报价中）。

### 4.3 参考价区间

**取整规则**：对外总报价 **百位四舍五入**（精确到百元）。

| 可定制开关 | 参考价计算 |
|------------|------------|
| **关** | 参考价下限 = 参考价上限 = 标配皮肤下的对外总报价（百位取整） |
| **开** | **下限** = 全部板材行均为标配颜色 → 对外总报价（百位取整） |
| | **上限** = 每一行分别取「同材质同厚度下最贵可选颜色」→ 汇总物料小计 × 1.10 → 百位取整 |

**上限计算示例**（可定制 = 开）：

- 行 A（免漆板 12mm，标配白色，可选：白 / 木色 / 砂面黑）→ 取三者板材对外价最高者  
- 行 B（海洋板 18mm，标配木色，可选：木色 / 白色）→ 取两者最高者  
- 骨架（型材 / 六通 / 五金）不变 → 汇总 → × 1.10 → 百位四舍五入  

> **不再使用**「总计 × 1.15」作为参考价区间规则。区间仅来自**皮肤色差**（可定制 = 开时）。

## 5. 角色与分工

| 角色 | 职责 |
|------|------|
| **录入员** | 上传 skp、选择场景；系统自动生成编号与名称；按分类上传图片、填写尺寸与 BOM |
| **清单兼审核** | 完整 BOM、报价口径、参考价与可定制开关核对；审核发布（小团队可一人兼任） |
| **审核员** | 按发布 checklist 质检 → 发布或退回 |

同一人可兼多职。录入员在 skp 上传并完成系统登记后，通常由同一人继续完成 BOM 与报价，**不建议**将「补全信息」与「清单报价」拆给不同人且无交接说明。

## 6. 协作流程

### 6.1 流程总览

```
录入员：新建模板 → 上传 skp（本地已简单命名）
    ↓
录入员：选择应用场景 → 系统自动生成编号、名称、负责人
    ↓                    并在 data/uploads/{模板编号}/ 创建目录、写入 skp
录入员：上传图片（实拍 / 效果图 / 渲染图）+ 尺寸 + BOM + 报价口径
    ↓
审核员：checklist 通过 → 已发布（审核单存档至 data/uploads/审核存档/）
    ↓
同步至飞书多维表格（模板主表）
    ↓
客户：打开飞书「画册视图」浏览方案
```

> **前提**：开始录入前，本地应已有一个**建模完成的 `.skp` 文件**，且文件名简洁可读（见 §6.4）。

### 6.2 状态说明

| 状态 | 含义 |
|------|------|
| 待清单深化 | skp 已登记，待补全图片、尺寸、BOM 与参考价 |
| 待审核 | BOM 与报价口径齐全，等待审核 |
| 已发布 | 已同步至飞书；客户可在画册视图中查看；**仅可改标签**，其它内容须先下架 |
| 已下架 | 暂停对外展示；可编辑后 **重新上架 → 待审核** |

登记 skp 后系统直接进入 **待清单深化**（已废除「待建模」）。审核员可将状态退回「待清单深化」，并在**内部备注**写明原因。

**已发布模板：** 详情「状态推进」仅 **下架** 按钮。除标签外不可修改（系统前后端均拦截）。修改流程：下架 → 编辑 → 重新上架 → 审核发布（版本 **+0.1**）；下架后若**更换 skp**（已有模型时再次上传）版本 **+1**。详见 [模板列表 §7](TEMPLATE-LIST-UPDATES.md)、[协作流程图 §6](WORKFLOW.md)。

### 6.3 步骤详解

#### 步骤 1：上传 skp 并登记（录入员）

录入时通常**已有成品 skp**，无需先填表单再建模。

1. 确认本地 skp 已用**简单名称**命名（见 §6.4），例如：`L型展架.skp`
2. 打开系统 → **新建模板**
3. **第一步：上传 skp 文件**（尚未生成正式模板编号）
4. **选择应用场景**（展厅 / 零售 / 活动 / 办公 / 其他）
5. 系统自动完成（**无需手工填写**）：
   - **模板编号**：如 `TPL-ZT-0001-L型展架`（规则见 §12）
   - **模板名称**：从 skp 文件名自动拾取（不含 `.skp` 扩展名）
   - **负责人**：当前登录录入人姓名
   - **可定制开关**：默认 **关**
6. 系统自动处理 skp 与目录（见 §12.3）：
   - 创建 **`data/uploads/{模板编号}/`**
   - 将 skp 重命名为 **`{模板编号}.skp`** 并写入该目录
7. 记录生成的**模板编号**，进入详情页继续补全（状态：**待清单深化**）

> **不再**向 `\\dsparti\home\Parti template model\` 复制 skp；所有模板资产统一存放在 `data/uploads/` 下。

#### 步骤 2：补全资料与清单（录入员 / 同一人）

1. **模板列表** → 点击编号进入详情
2. 在 **`data/uploads/{模板编号}/`** 对应分类下上传图片（见 §12.3）：
   - **实拍照片**：可选，可多张
   - **效果图**：必填，≥ 1 张
   - **渲染图**：必填，≥ 1 张
3. 确认 **封面**已由系统按规则自动选取（有实拍 → 用实拍；无实拍 → 用效果图）；可在详情 **基本信息** 用 **缩略图下拉** 更换封面来源（选中即保存）
4. 填写外形尺寸：**宽 / 深 / 高（mm）**
5. 录入 **结构 BOM**：
   - 型材：按根录入**颜色**、长度（inch）与数量（颜色在单价库维护，银色/木色等价）
   - 六通 / 五金：从单价库选取，数量与 skp 一致
6. 录入 **板材 BOM**（一行或多行）：
   - 每行：材质类型、厚度、**标配颜色**、长宽（inch）、数量
   - 多材质 / 多厚度时分行填写，不可合并
7. 填写 **报价口径**（含/不含安装、运输、灯带等）
8. 填写 **一句话卖点**（将展示在飞书画册视图，必填）
9. 在详情 **标签** 区打标：输入 `#` 搜索标签库或创建新标签（见 [标签库说明](TAG-LIB-UPDATES.md)）；也可点常用 Top5
10. 核对 **参考价**（百位取整；可定制关 → 单一价位）；按需开启 **可定制**
11. 在 **基本信息** 分组中保存（名称 / 尺寸 / 卖点 / 报价 / 参考价 / 可定制等）
12. 点击 **→ 待审核**

> **模板列表** 可按标签筛选（多标签取**交集**）：工具栏 `#添加筛选`，或从 **标签库** 字云点击追加（见 [标签库说明](TAG-LIB-UPDATES.md) §3、[模板列表 §9](TEMPLATE-LIST-UPDATES.md)）。

> **不再**在详情填写 **皮肤选配说明**、**询单表单链接**。历史模板若已有 `panel_note`，详情以只读块展示；图册 PDF 有内容仍会输出。

> skp 已在步骤 1 写入模板目录，**本步骤无需再次上传 skp**，除非更换模型版本（升版时见 §12.3）。

#### 步骤 3：审核发布（审核员）

审核在 catalog **审核页面**内完成，不使用纸质或外部打印勾选表。

1. 模板进入 **待审核** 后，审核员打开该模板的 **审核页**
2. 对照 skp 与 BOM，在页面上**逐项勾选**发布 checklist（见 §7.2）
3. 填写 **审核备注**（可选）；若退回，必填 **退回原因**
4. 点击 **通过并发布** 或 **退回**
5. **审核通过时，系统自动**：
   - 将模板状态更新为 **已发布**
   - 生成 **审核单图片**（含勾选结果、审核员、审核时间、模板编号等）
   - 在 **审核记录台账 CSV** 中追加一行（编号、通过日期、审核人、备注等，见 §7.4）
   - 将审核单图片与台账更新 **存档至** `data/uploads/审核存档/`（路径见 §12.4）
6. **不需要**填写单模板飞书 Doc 链接

> 退回时：不生成审核单、不写入审核存档；模板回到「待清单深化」，录入员修改后重新提交审核。

#### 步骤 4：同步飞书并发给客户

**对外主渠道是飞书多维表格**，不在微信里逐条粘贴方案文字。

**4.1 发布后同步（录入员 / 销售）**

1. 模板状态变为 **已发布** 后，在 catalog 侧边栏 **导出主表 CSV**
2. 将对应行更新至飞书 **模板主表**（每发布 1 个模板同步 1 次；每周可全量核对）
3. 在飞书主表中补充 **封面** 附件（与 catalog 自动选取的封面一致；CSV 不含图片，见 §9.3）
4. 确认该行在 **对外-画册** 视图中显示正常

**4.2 发给客户（销售）**

1. 打开飞书多维表格 → 切换至 **对外-画册** 视图（见 §9.2）
2. 点击视图右上角 **分享**，生成客户可读链接（建议：互联网只读或组织内只读，按客户类型设置）
3. 将 **画册视图链接** 发给客户（微信 / 邮件 / 报价单附件说明均可）
4. 客户在画册中可直观浏览：
   - **封面图片**
   - **应用场景**
   - **参考价格**（下限 – 上限，或单一价位）
   - **一句话卖点**
   - **标签**（支持筛选）
   - 以及模板名称、外形尺寸等（按视图配置）

**4.3 客户询单后（销售 / 设计）**

1. 客户从画册中选中意向方案，告知 **模板编号** 或名称
2. 内部基于 skp + BOM 出**正式报价**（参考价不等于成交价）
3. 如需单独说明报价口径，可引用 catalog 中的 **报价口径** 字段内容

> **对外交付**：catalog **模板图册**下载展示图 PNG 与**方案手册** PDF（对外脱敏 BOM）；内部生产/采购用 **下载清单** CSV（含单价与供应商，须导出权限，可批量）。飞书多维表格画册视图仍为对外主渠道之一。详见 [模板图册说明](GALLERY-UPDATES.md)、[协作流程图 §8.1](WORKFLOW.md)。

### 6.4 skp 命名规范

#### 上传前（本地文件）

- 使用**简短、可读的中文或英文名称**，描述方案类型即可
- 示例：`L型展架.skp`、`宠物跳台三层.skp`、`市集摊位单联.skp`
- 避免：`未命名.skp`、`模型1.skp`、过长路径名或含特殊字符

此名称将用于：

- 自动拾取为**模板名称**
- 参与生成**模板编号**中间段（见 §12.1）

#### 上传后（系统自动）

登记完成后，skp 位于：

```
data/uploads/{模板编号}/{模板编号}.skp
```

示例：本地 `L型展架.skp` → `data/uploads/TPL-ZT-0001-L型展架/TPL-ZT-0001-L型展架.skp`

#### 版本号（系统自动）

- 格式：`v主版本.次版本`（如 `v1.0`、`v1.1`、`v2.0`）
- **首次审核发布**：将登记时的版本规范为 `v1.0` 等
- **再次审核发布**（曾发布过的模板）：次版本 **+0.1**
- **更换 skp 模型**（模板已有 skp 时再次上传）：主版本 **+1**，次版本归零
- 详情页版本字段**只读**，不可手工填写
- 模板编号不变；须先**下架**再改 skp/BOM，再重新上架审核

## 7. 发布齐全标准与审核页

### 7.1 发布齐全标准

以下任一项缺失，**不得通过审核、不得发布**。

| # | 检查项 | 要求 |
|---|--------|------|
| 1 | 模板编号、名称、应用场景 | 必填（skp 上传后系统自动生成编号与名称） |
| 2 | 负责人 | 必填（系统自动拾取录入人） |
| 3 | 外形尺寸 宽 × 深 × 高 | 必填（mm） |
| 4 | 封面 | 已由系统按 §12.3 规则自动选取 |
| 5 | 效果图 | ≥ 1 张（必填） |
| 6 | 渲染图 | ≥ 1 张（必填） |
| 7 | 实拍照片 | 可选 |
| 8 | skp 源文件 | 必填；位于 `data/uploads/{模板编号}/{模板编号}.skp` |
| 9 | 结构 BOM | 型材 / 六通 / 五金与 skp 一致 |
| 10 | 板材 BOM | 一行或多行，标配颜色齐全 |
| 11 | 报价口径 | 已写 |
| 12 | 参考价 | 百位取整正确（可定制关：单价；可定制开：区间） |
| 13 | 一句话卖点 | 必填（飞书画册展示） |
| 14 | 标签 | 建议填写（飞书筛选） |
| 15 | 可定制开关 | 新模板默认 **关**；开时参考价区间与单价库异色项一致 |

> **飞书主表同步**在发布后 **步骤 4** 执行，不属于审核页勾选范围。

### 7.2 审核页操作（系统内）

审核员在 catalog **审核页**对「待审核」模板操作，界面包含：

| 区域 | 内容 |
|------|------|
| **模板摘要** | 编号、名称、应用场景、负责人、参考价、封面预览 |
| **勾选清单** | §7.1 第 1–15 项，逐项 checkbox（与 `AUDIT_CHECKLIST` 同步） |
| **审核信息** | 审核员（自动拾取登录人）、审核时间（自动生成） |
| **审核备注** | 选填 |
| **退回原因** | 退回时必填 |
| **操作按钮** | 「通过并发布」/ 「退回至待清单深化」 |

**通过并发布** 后系统自动：

1. 记录勾选结果与审核信息  
2. 生成 **审核单图片**（PNG，便于查阅与留痕）  
3. 向 **审核记录台账 CSV** 追加一行审核数据（§7.4）  
4. 将审核单图片写入 **`data/uploads/审核存档/`**（§12.4）  
5. 更新状态为 **已发布**

> 审核存档 **不包含** 模板主表 CSV 或 BOM CSV；主表 / BOM 仍通过 catalog 常规模块导出，用于飞书同步（步骤 4）。

### 7.3 审核单图片内容

审核单图片为只读存档，建议包含：

- 模板编号、名称、应用场景、负责人  
- 外形尺寸、参考价区间  
- 全部 checklist 勾选状态（✓ / ✗）  
- 审核员、审核时间  
- 审核结论：**通过**  
- 审核备注（若有）

文件名：`{模板编号}_审核单_{YYYYMMDD_HHmmss}.png`

### 7.4 审核记录台账 CSV

审核通过后，系统向 **`data/uploads/审核存档/审核记录.csv`** 追加一行。该文件是**审核单数据表**（台账），用于检索「谁在何时通过了哪个模板」，**不是**模板主表或 BOM 导出。

**文件路径**：

```
data/uploads/审核存档/审核记录.csv
```

**写入规则**：

| 规则 | 说明 |
|------|------|
| 文件不存在 | 首次审核通过时自动创建，并写入表头 |
| 每次审核通过 | 在文件末尾 **追加一行**（不覆盖历史） |
| 同一模板再次发布 | 产生新一行（带新的通过日期与审核单图片路径） |
| 审核退回 | **不写入** |

**建议字段（列）**：

| 列名 | 说明 | 示例 |
|------|------|------|
| 模板编号 | 唯一标识 | `TPL-ZT-0001-L型展架` |
| 模板名称 | 发布时名称 | `L型展架` |
| 应用场景 | 发布时场景 | `展厅` |
| 通过日期 | 审核通过时间 | `2026-05-27 14:30:52` |
| 审核人 | 登录审核员姓名 | `李四` |
| 审核备注 | 审核页填写的备注 | `BOM 已核对，参考价合理` |
| 参考价下限 | 通过时参考价 | `3200` |
| 参考价上限 | 通过时参考价 | `3200` |

> 实现阶段可在 catalog 数据库同步保存相同字段；`data/uploads/审核存档/审核记录.csv` 作为可独立打开的归档副本，便于 Excel / 飞书导入查阅。

## 9. 飞书多维表格 · 对外画册

### 9.1 系统分工

```
catalog（内部主系统：录入 + BOM + 算价 + 审核）
    ↓ 导出 CSV，更新飞书「模板主表」
飞书多维表格
    ├── 内部视图：全部 / 待办 / 审核队列（团队协作用）
    └── 对外-画册视图：分享链接给客户浏览方案
```

- **catalog** 负责编辑与算价；**飞书** 负责**对客户的方案展示与筛选**。
- **不需要**为每个模板单独维护飞书 Doc 链接。
- 客户收到的是 **一个多维表格的画册视图分享链接**，可在手机/电脑浏览全部已发布方案。

表格搭建与字段类型详见 [FEISHU-SETUP.md](FEISHU-SETUP.md)。

### 9.2 对外-画册视图配置

在飞书 **模板主表** 中新建视图：**对外-画册**（类型：画册视图）。

| 配置项 | 建议设置 |
|--------|----------|
| **筛选** | 状态 = 已发布 |
| **封面** | 封面（附件或 URL 字段） |
| **标题** | 模板名称 |
| **副标题 / 描述** | 一句话卖点 |
| **展示字段** | 应用场景、参考价下限、参考价上限、标签、宽×深×高 |
| **隐藏字段** | 内部备注、skp、BOM 明细、对内成本、负责人（按需） |
| **分享权限** | 互联网只读 或 组织内只读（按客户类型） |

**客户看到的核心信息**：

| 画册展示项 | 对应 catalog / 飞书字段 |

| 封面图片 | 封面 |

### 9.3 同步节奏与封面图片

| 时机 | 操作 |
|------|------|
| 每发布 1 个模板 | 导出主表 CSV → 更新飞书对应行 → 上传封面图 |
| 每周 | 全量导出核对，确保已发布方案与 catalog 一致 |

**封面说明**：CSV 导出**不含图片文件**。catalog 已按 §12.3 自动选取封面；同步飞书时：

实拍照片、效果图、渲染图、skp、BOM 明细**不放入**对外画册视图（画册仅展示封面）。

#### 目录结构

每个模板独占一个以 **模板编号** 命名的文件夹：

```
data/uploads/
├── TPL-ZT-0001-L型展架/
│   ├── TPL-ZT-0001-L型展架.skp      ← 模型源文件
│   ├── 实拍照片/                     ← 可选，可多张
│   │   └── …

│   │   └── …
│   └── 渲染图/                       ← 必填 ≥ 1 张
│       └── …
├── TPL-LS-0002-宠物跳台/
│   └── …
└── 审核存档/                         ← 见 §12.4
    ├── 审核记录.csv
    └── …
```

#### 图片三类与封面规则

| 类型 | 是否必填 | 用途 |
|------|----------|------|
| **实拍照片** | 否 | 真实产品/现场照片；有则优先作封面 |
| **效果图** | 是（≥ 1 张） | 方案示意、排版效果；无实拍时作封面 |
| **渲染图** | 是（≥ 1 张） | 三维渲染展示；不参与封面自动选取 |

**封面自动选取规则**（系统执行，无需单独上传「封面」类文件）：

```
若 实拍照片/ 下有图片 → 封面 = 首张实拍照片（或用户手动指定的实拍）
否则若 效果图/ 下有图片 → 封面 = 首张效果图（或用户手动指定的效果图）
否则 → 不满足发布条件
```

- 详情 **基本信息** 可用 **缩略图下拉** 更换封面来源（仍须来自实拍或效果图；选中即保存）。
- 渲染图仅作展示归档，**不作为**封面候选。

| 阶段 | 路径 |

| 上传前（本地） | 简短名称，如 `L型展架.skp` |
| 登记后 | `data/uploads/{模板编号}/{模板编号}.skp` |

| SOP 规则 | 当前系统 | 目标行为 |
