# 飞书多维表格搭建指南

本地模板库负责录入、协作、算价；飞书负责 **对外分享、权限、移动端查看**。通过 CSV 同步两者。

## 第一步：创建 Base

1. 飞书 → **多维表格** → 新建「MR2525 方案模板库」
2. 创建三张表：
   - `模板主表`
   - `报价底表`
   - `单价库`

## 第二步：导入 CSV

使用本项目 `data/feishu/` 下的文件：

| 文件 | 导入到 |
|------|--------|
| `单价库.csv` | 单价库 |
| `模板主表-导入模板.csv` | 模板主表 |
| `报价底表-导入模板.csv` | 报价底表 |

导入后调整字段类型：

### 模板主表字段类型

| 字段 | 类型 |
|------|------|
| 模板编号 | 文本（唯一） |
| 宽mm / 深mm / 高mm | 数字 |
| 参考价下限 / 参考价上限 | 数字 / 货币 |
| 标签 / 变价因素 | 多选（手动拆分为多选列亦可） |
| 状态 | 单选：待建模、待清单报价、待审核、已发布、已下架 |
| 详情链接 / 询单链接 | URL |
| 封面 / 效果图 | 附件（本地导出 CSV 不含附件，需在飞书手动补或改用本地链接） |

### 报价底表

- **所属模板编号**：关联 → 模板主表（按模板编号匹配）
- **小计**：公式 `数量 * 单价`

### 单价库

- 供 BOM 填行时人工参照；或在飞书用查找引用关联

## 第三步：创建视图

| 视图 | 筛选 | 用途 |
|------|------|------|
| 内部-全部 | 无 | 团队编辑 |
| 内部-待办 | 负责人=我 且 非已发布 | 个人任务 |
| 对外-已发布 | 状态=已发布 | 分享此视图链接给客户筛选 |
| 审核队列 | 状态=待审核 | 审核员 |

**注意**：对外视图不要包含「内部备注、物料成本明细单价、skp」等列。

## 第四步：新建表单（可选，与本地库二选一或并行）

若同事更习惯飞书表单：

1. 基于 **模板主表** 创建表单
2. 只收集：名称、场景、标签、面板类型、负责人、一句话描述
3. 飞书自动化：提交后写入「待建模」

**推荐**：日常协作用 **本地 Web 库**（自动编号 + BOM + 文件上传更完整），定期 **导出 CSV 覆盖/增量同步** 飞书主表。

## 第五步：同步节奏

| 频率 | 操作 |
|------|------|
| 每完成 1 个模板发布 | 导出 CSV，在飞书更新对应行 |
| 每周 | 全量导出主表 + BOM，核对飞书 |
| 单价变更 | 两边单价库同步 |

导出入口：本地系统左侧 **导出主表 CSV** / **导出 BOM CSV**。

## 第六步：详情 Doc 模板

审核发布后，在飞书用 Doc 模板创建详情页，结构建议：

```
# {模板编号} {模板名称}

参考价：¥{下限} – ¥{上限}
{报价口径}

## 方案说明
{一句话描述}

## 尺寸
{宽} × {深} × {高} mm

## 面板
{面板说明}

## 效果图
（粘贴图片）

## 询单
{询单表单链接}
```

将 Doc 链接填回本地详情页 **飞书详情 Doc 链接** 字段，下次导出 CSV 会带上。

## 第七步：权限

- 内部 Base：设计/销售团队编辑
- 对外视图：仅特定列，链接可设为组织内或互联网只读
- skp 与成本：仅内部 Base，勿放入公开 Doc

## 后期迁自建站

本地库已含 `slug`、参考价、状态；迁站时导出 `catalog.db` 或 CSV 即可。
